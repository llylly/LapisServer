Write API Specification Script

Following the specification documentation PDF, or Examples, you can document your APIs.
The specification is extended from Swagger (OpenAPI), so it's also OK to migrate from there. Although you will miss our extended features.
The code editor is in left column of Front Page.

Load Script From Your Computer

You can load the script from local file system.
After selected your script file, remember to click "Load" to load it to the code editor.

Parse

Choose the format of script. We support JSON, YAML and XML format script. Then click "Parse" to get the parse result.
The parse process contains two stages: Document Parse, API Parse.
In Document Parse, we check the format correctness. In API Parse, we check the meanings of each fields.
When error detected, they are demonstrated in the right column.


Read Parse Result

The result contains 3 parts: basic info, API details, and request data generation.
Basic Info:
The basic info section demonstrates essential info of the script.
API Detail:
Each API is shown in a panel. Click the API name, you can unfold the panel. The parameters and responses panel in it can also be unfolded for detail.

Request Data Generation:
In Data Generation panel, select an API, then click "Generate Request Data" to get an example of request data generated.

Register and Store Your Document

Register, then you will get a repository to store and retrieve your documents conveniently.
The "Register" button is in the right side of top navigation bar.

After login, in the left side, "My Files" column will show. Click each name, the saved document will be loaded to the code editor.
Hover over the names, the "Rename" and "Delete" button will show.

To save the modified version, at first, you can click the filename at the top of code editor to change document name.
After that, click "Save" button below the "Parse" button to save. As the result, the document will show immediately in the left column.
